# 🔧 Opera 手动配置指南

## ⚠️ 问题说明

Opera 的所有自动优化都失败了，原因：
1. Opera 使用自己的扩展商店，不兼容 Chrome Web Store
2. Opera 的企业策略支持有限
3. Opera 的配置文件格式与 Chrome 不完全兼容

---

## ✅ 解决方案：使用 Opera 内置功能 + 手动配置

Opera 自带很多隐私保护功能，不需要安装太多扩展！

---

## 🚀 快速修复

### 步骤1：运行修复脚本

```powershell
# 以管理员身份运行
.\FIX_OPERA_COMPLETE.ps1
```

这个脚本会：
- ✅ 创建正确的配置文件
- ✅ 配置企业策略（尽可能）
- ✅ 创建 Opera 启动脚本
- ✅ 自动启动 Opera 供你手动配置

### 步骤2：手动配置 Opera

启动 Opera 后，按照以下步骤配置：

---

## 📋 详细配置步骤

### 1. 基本设置

**打开设置：** 点击左上角 Opera 图标 → 设置（或按 `Alt+P`）

#### 1.1 启动页面
```
设置 → 浏览器 → 启动时
选择：打开特定页面或一组页面
点击"添加新页面"
输入：about:blank
点击"添加"
```

#### 1.2 显示书签栏
```
设置 → 浏览器 → 用户界面
勾选：显示书签栏
```

#### 1.3 主页设置
```
设置 → 浏览器 → 主页
选择：自定义地址
输入：about:blank
```

---

### 2. 隐私和安全设置

#### 2.1 启用广告拦截（内置！）
```
设置 → 基本设置 → 广告拦截
勾选：启用广告拦截
```

**说明：** Opera 内置广告拦截器，效果类似 uBlock Origin，无需安装扩展！

#### 2.2 启用追踪保护（内置！）
```
设置 → 隐私和安全 → 追踪保护
勾选：启用追踪保护
```

**说明：** Opera 内置追踪保护，效果类似 Privacy Badger！

#### 2.3 Cookie 设置
```
设置 → 隐私和安全 → Cookie 和其他网站数据
选择：阻止第三方 Cookie
```

#### 2.4 禁用位置访问
```
设置 → 隐私和安全 → 网站设置 → 位置
选择：不允许网站请求您的位置
```

#### 2.5 禁用通知
```
设置 → 隐私和安全 → 网站设置 → 通知
选择：不允许网站发送通知
```

#### 2.6 禁用 WebRTC（重要！）
```
在地址栏输入：opera://flags
搜索：WebRTC
找到：WebRTC IP handling policy
选择：Disable non-proxied UDP
重启 Opera
```

---

### 3. 禁用不需要的功能

#### 3.1 禁用 Opera VPN
```
设置 → 隐私和安全 → VPN
取消勾选：启用 VPN
```

**重要：** 我们使用 Clash 代理，不需要 Opera VPN！

#### 3.2 禁用新闻
```
设置 → 基本设置 → 新闻
取消勾选：在新标签页上显示新闻
```

#### 3.3 禁用侧边栏
```
设置 → 浏览器 → 侧边栏
取消勾选所有不需要的侧边栏项目
```

#### 3.4 禁用遥测
```
设置 → 隐私和安全 → 隐私
取消勾选：帮助改进 Opera
取消勾选：发送使用情况统计信息
```

---

### 4. 安装必要扩展（仅1个）

Opera 内置了广告拦截和追踪保护，只需要安装 WebRTC 防护扩展：

#### 4.1 打开扩展页面
```
按 Ctrl+Shift+E
或点击：菜单 → 扩展 → 扩展
```

#### 4.2 安装 WebRTC Leak Shield
```
1. 点击"获取扩展"
2. 搜索：WebRTC Leak Shield
3. 点击"添加到 Opera"
4. 确认安装
```

**说明：** 这是唯一必须安装的扩展，防止 WebRTC 泄露真实 IP！

---

### 5. 代理设置

Opera 会自动使用系统代理，但我们通过启动参数指定：

#### 5.1 使用启动脚本
```powershell
# 运行 Opera 启动脚本
.\LAUNCH_OPERA.ps1
```

这个脚本会自动配置：
- ✅ 代理服务器：socks5://127.0.0.1:7890
- ✅ 用户数据目录：C:\BrowserProfiles\Opera
- ✅ 禁用 WebRTC
- ✅ 禁用后台网络

## 🔧 故障排除

### Q1: 广告拦截不生效

**解决方法：**
```
1. 设置 → 基本设置 → 广告拦截
2. 确保"启用广告拦截"已勾选
3. 点击"管理列表"
4. 确保 EasyList 已启用
```

### Q2: WebRTC 仍然泄露

**解决方法：**
```
1. 确保 WebRTC Leak Shield 扩展已安装
2. opera://flags → WebRTC IP handling policy → Disable non-proxied UDP
3. 重启 Opera
```

### Q3: 代理不生效

**解决方法：**
```
1. 确保 Clash 正在运行
2. 使用 .\LAUNCH_OPERA.ps1 启动（不要直接双击）
3. 访问 https://ipinfo.io 检查 IP
```

### Q4: 启动时不显示空白页

**解决方法：**
```
1. 设置 → 浏览器 → 启动时
2. 选择"打开特定页面或一组页面"
3. 删除所有现有页面
4. 添加 about:blank
```
