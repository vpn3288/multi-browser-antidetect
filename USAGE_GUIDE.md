## 🚀 快速开始

### 第一步：运行部署脚本

1. 以**管理员身份**打开 PowerShell
2. 进入项目目录
3. 运行部署脚本：

```powershell
cd multi-browser-antidetect
.\DEPLOY_8_BROWSERS.ps1
```

脚本会自动：
- ✅ 安装缺失的浏览器
- ✅ 配置所有隐私和安全设置
- ✅ 为每个浏览器生成唯一的真实指纹
- ✅ 创建启动脚本

### 第二步：配置 Clash 代理

确保每个浏览器使用**不同的美国IP地址**：

```yaml
# Clash 配置示例
proxy-groups:
  - name: "Chrome-7891"
    type: select
    proxies: ["US-NewYork-1"]
    
  - name: "Firefox-7893"
    type: select
    proxies: ["US-LosAngeles-2"]
    
  # ... 为每个端口配置不同的美国IP
```

### 第三步：启动浏览器

```batch
# 启动所有浏览器
C:\BrowserProfiles\Launch_All.bat

# 或单独启动
C:\BrowserProfiles\Launch_Chrome.bat
```

## ❓ 常见问题

### Q1: 为什么不完全禁用WebGL？

**A:** 完全禁用WebGL会让你看起来更可疑，因为：
- 真实用户很少禁用WebGL
- 很多网站需要WebGL才能正常工作
- 禁用WebGL反而是一个明显的"我在隐藏什么"的信号

我们的方案是**保留WebGL但随机化GPU信息**，这样既能正常使用网站，又能保护隐私。

### Q2: 为什么不使用 `privacy.resistFingerprinting`？

**A:** Firefox的 `privacy.resistFingerprinting` 会让所有用户看起来一样：
- 所有人都是1920x1080分辨率
- 所有人都是相同的Canvas指纹
- 所有人都禁用了WebGL

这反而让你更容易被识别为"使用隐私保护的用户"。

我们的方案是**让每个浏览器有唯一但真实的指纹**，就像真实的不同用户。

### Q3: Canvas噪声会影响显示效果吗？

**A:** 不会。我们添加的噪声非常微小（0.0001级别），肉眼完全看不出来，但足以改变指纹哈希值。

### Q4: 为什么要使用不同的屏幕分辨率？

**A:** 如果8个浏览器都是相同的分辨率，网站可能会怀疑它们来自同一台机器。

我们使用8种不同的真实常见分辨率，让每个浏览器看起来来自不同的设备。

### Q5: 指纹会每次都变吗？

**A:** 不会。每个浏览器的指纹是**稳定的**：
- 同一浏览器每次启动指纹相同
- 不同浏览器之间指纹不同
- 这样既保护隐私，又不会因为指纹频繁变化而被怀疑

### Q6: 需要安装浏览器扩展吗？

**A:** 不需要。所有优化都通过：
- 注册表策略（Chromium系）
- user.js配置（Firefox系）
- 启动参数

不需要安装任何扩展，避免扩展本身成为指纹特征。

## 🎓 高级技巧

### 技巧1: 模拟真实用户行为

不要：
- ❌ 机械化地每隔固定时间访问网站
- ❌ 总是按相同顺序点击
- ❌ 从不移动鼠标就直接点击
- ❌ 访问速度过快（人类不可能那么快）

应该：
- ✅ 随机化访问时间
- ✅ 模拟鼠标移动
- ✅ 偶尔停顿、滚动页面
- ✅ 像真人一样浏览

### 技巧2: 使用不同的账号信息

每个浏览器应该：
- ✅ 使用不同的邮箱
- ✅ 使用不同的用户名
- ✅ 使用不同的密码
- ✅ 不要在多个浏览器登录同一账号

### 技巧3: 定期清理Cookie

虽然我们已经阻止第三方Cookie，但仍建议：
- 每周清理一次Cookie
- 或者使用浏览器的"退出时清理"功能

### 技巧4: 使用不同的书签

让每个浏览器的书签看起来不同：
- Chrome: 科技新闻、编程网站
- Firefox: 开源社区、技术博客
- Edge: 微软产品、办公软件
- 等等...

这样即使有人检查你的浏览器，也看起来像不同的人。

### 技巧5: 调整浏览器窗口大小

不要总是全屏或固定大小，偶尔调整窗口大小，让它看起来更自然。

## 🔧 故障排除

### 问题1: 浏览器无法启动

**可能原因：**
- 浏览器未正确安装
- 配置目录权限问题

**解决方法：**
```powershell
# 重新运行部署脚本
.\DEPLOY_8_BROWSERS.ps1
```

### 问题2: 代理不工作

**检查清单：**
- ✅ Clash 是否正在运行？
- ✅ 代理端口是否正确配置（7891-7898）？
- ✅ 防火墙是否阻止了连接？

**测试代理：**
```powershell
# 测试端口是否开放
Test-NetConnection -ComputerName 127.0.0.1 -Port 7891
```

### 问题3: 仍然检测到真实IP

**可能原因：**
- WebRTC泄漏
- DNS泄漏
- 代理配置错误

**解决方法：**
1. 访问 https://browserleaks.com/webrtc 检查WebRTC
2. 访问 https://www.dnsleaktest.com/ 检查DNS
3. 确保Clash配置了DNS

### 问题4: Canvas指纹相同

**可能原因：**
- 指纹保护脚本未加载

**解决方法：**
```powershell
# 检查脚本是否存在
Get-ChildItem C:\BrowserProfiles\*\fingerprint_protection.js

# 如果不存在，重新运行部署
.\DEPLOY_8_BROWSERS.ps1
```

### 问题5: 书签栏不显示

**解决方法：**

**Chromium系浏览器：**
- 按 `Ctrl + Shift + B` 切换书签栏

**Firefox系浏览器：**
- 右键点击工具栏 → 勾选"书签工具栏"

如果还是不显示，重新运行优化脚本：
```powershell
.\OPTIMIZE_ARCHITECTURE.ps1
```

### 问题6: 新标签页不是空白页

**解决方法：**

重新运行部署脚本，或手动设置：

**Chrome/Edge/Brave:**
- 设置 → 启动时 → 打开特定网页 → 添加 `about:blank`

**Firefox:**
- 设置 → 主页 → 主页和新窗口 → 选择"空白页"
