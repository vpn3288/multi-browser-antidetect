# 🚀 快速开始指南（中国用户）

## 📋 前提条件

- ✅ Windows 11 系统
- ✅ PowerShell 7.6.1+
- ✅ Clash Verge（或其他 Clash 客户端）
- ✅ 有效的代理节点（美国、日本、新加坡、英国、德国、加拿大、澳大利亚）

## 🎯 5分钟快速部署

### 第1步：下载项目

```powershell

```

或者直接下载 ZIP：
https://github.com/vpn3288/multi-browser-antidetect/archive/refs/heads/master.zip

### 第2步：配置 Clash

1. **打开 Clash Verge**
2. **编辑配置文件**（设置 → 配置文件 → 编辑）
3. **复制以下内容到配置文件末尾**：

```yaml
# 为每个浏览器配置独立监听端口
listeners:
  - name: chrome-us
    type: mixed
    port: 7891
    proxy: 🇺🇸 美国节点  # 替换为你的实际节点名称
    
  - name: firefox-jp
    type: mixed
    port: 7892
    proxy: 🇯🇵 日本节点  # 替换为你的实际节点名称
    
  - name: edge-sg
    type: mixed
    port: 7893
    proxy: 🇸🇬 新加坡节点  # 替换为你的实际节点名称
    
  - name: brave-uk
    type: mixed
    port: 7894
    proxy: 🇬🇧 英国节点  # 替换为你的实际节点名称
    
  - name: opera-de
    type: mixed
    port: 7895
    proxy: 🇩🇪 德国节点  # 替换为你的实际节点名称
    
  - name: vivaldi-ca
    type: mixed
    port: 7896
    proxy: 🇨🇦 加拿大节点  # 替换为你的实际节点名称
    
  - name: librewolf-au
    type: mixed
    port: 7897
    proxy: 🇦🇺 澳大利亚节点  # 替换为你的实际节点名称

# DNS 配置（防止 DNS 泄露）
dns:
  enable: true
  enhanced-mode: fake-ip
  nameserver:
    - https://1.1.1.1/dns-query
    - https://8.8.8.8/dns-query
```

4. **保存配置**
5. **重启 Clash Verge**

## 🔧 常见问题

### Q1: 代理端口连接失败

**原因**：Clash 未启动或配置错误

**解决方法**：
1. 确保 Clash Verge 正在运行
2. 检查配置文件中的节点名称是否正确
3. 重启 Clash Verge

### Q2: WebRTC 泄露真实 IP

**原因**：WebRTC 未禁用

**解决方法**：
1. 安装 WebRTC Leak Shield 扩展
2. 或在浏览器启动参数中添加 `--disable-webrtc`

### Q3: 时区显示中国时间

**原因**：时区注入脚本未生效

**解决方法**：
1. 检查 `timezone_injector.js` 是否正确注入
2. 手动在浏览器设置中修改时区

### Q4: 扩展未自动安装

**原因**：企业策略未生效

**解决方法**：
1. 以管理员身份重新运行 `INSTALL_EXTENSIONS_OPTIMIZED.ps1`
2. 重启浏览器
3. 首次启动时扩展会自动下载

### Q5: 浏览器启动慢

**原因**：配置文件过大或硬件性能不足

**解决方法**：
1. 使用 `LAUNCH_BROWSERS_FAST.ps1` 并行启动
2. 关闭不需要的浏览器
3. 清理浏览器缓存

## 🎯 使用建议

### 1. 日常使用

- **访问国际网站**：使用 Chrome (US)
- **观看日本内容**：使用 Firefox (JP)
- **玩国际游戏**：使用 Edge (SG) 或 Brave (UK)
- **下载文件**：使用 Opera (DE)
- **开发测试**：使用 Vivaldi (CA)
- **极致隐私**：使用 LibreWolf (AU)

### 2. 安全建议

- ✅ 不要在所有浏览器中登录同一个账号
- ✅ 每个浏览器使用不同的用户名和密码
- ✅ 定期清理浏览器缓存和 Cookie
- ✅ 定期更新浏览器和扩展
- ✅ 定期更换代理节点

### 3. 性能优化

- ✅ 只启动需要的浏览器
- ✅ 关闭不使用的标签页
- ✅ 定期清理浏览器缓存
- ✅ 使用 SSD 存储浏览器配置文件
