## 🚀 快速开始

### 方法一：一键配置所有浏览器（推荐）

1. **以管理员身份打开 PowerShell**
   - 右键点击 PowerShell 图标
   - 选择"以管理员身份运行"

2. **进入项目目录**
   ```powershell
   cd C:\path\to\multi-browser-antidetect
   ```

3. **运行统一配置脚本**
   ```powershell
   .\CONFIGURE_ALL_BROWSERS.ps1
   ```

4. **按照提示选择配置模式**
   - 选项 1：配置所有已安装的浏览器（推荐）
   - 选项 2：选择要配置的浏览器
   - 选项 3：退出

5. **等待配置完成**
   - 脚本会自动检测已安装的浏览器
   - 逐个配置每个浏览器
   - 显示配置结果

### 方法二：单独配置每个浏览器

如果你只想配置某个浏览器，可以单独运行对应的配置脚本：

```powershell
# Chrome
.\CHROME_ENTERPRISE_CONFIG.ps1

# Chromium
.\CHROMIUM_ENTERPRISE_CONFIG.ps1

# Edge
.\EDGE_ENTERPRISE_CONFIG.ps1

# Firefox
.\FIREFOX_ENTERPRISE_CONFIG.ps1

# Brave
.\BRAVE_ENTERPRISE_CONFIG.ps1

# Vivaldi
.\VIVALDI_ENTERPRISE_CONFIG.ps1

# LibreWolf
.\LIBREWOLF_ENTERPRISE_CONFIG.ps1

# Opera
.\OPERA_AUTO_CONFIG.ps1
```

## ✅ 验证配置

### 1. 验证企业策略

**Chromium 系浏览器（Chrome, Chromium, Edge, Brave, Vivaldi, Opera）：**
- 启动浏览器
- 在地址栏输入：`chrome://policy`（或 `edge://policy`, `brave://policy` 等）
- 查看已应用的策略列表

**Firefox 系浏览器（Firefox, LibreWolf）：**
- 启动浏览器
- 在地址栏输入：`about:policies`
- 查看已应用的策略列表

### 2. 验证扩展安装

**Chromium 系浏览器：**
- 在地址栏输入：`chrome://extensions`
- 查看已安装的扩展

**Firefox 系浏览器：**
- 在地址栏输入：`about:addons`
- 查看已安装的扩展

### 3. 运行验证脚本

```powershell
# Chrome
.\VERIFY_CHROME_CONFIG.ps1

# Opera
.\VERIFY_OPERA_CONFIG.ps1
```

## 🎮 启动浏览器

配置完成后，使用启动脚本启动浏览器（会自动配置代理）：

### 使用默认配置

```powershell
# 使用默认配置文件和代理端口 7890
.\LAUNCH_CHROME.ps1
.\LAUNCH_CHROMIUM.ps1
.\LAUNCH_EDGE.ps1
.\LAUNCH_FIREFOX.ps1
.\LAUNCH_BRAVE.ps1
.\LAUNCH_VIVALDI.ps1
.\LAUNCH_LIBREWOLF.ps1
.\LAUNCH_OPERA.ps1
```

### 自定义配置

```powershell
# 自定义配置文件和代理端口
.\LAUNCH_CHROME.ps1 -ProfileName "Profile1" -ProxyPort 7891
.\LAUNCH_FIREFOX.ps1 -ProfileName "work" -ProxyPort 7892
```

**参数说明：**
- `-ProfileName`：浏览器配置文件名称（用于多账号）
- `-ProxyPort`：代理端口（默认 7890）

## ⚠️ 重要注意事项

### 1. 管理员权限
**所有配置脚本都需要以管理员身份运行**，因为需要修改注册表。

### 2. 浏览器重启
配置完成后，**必须重启浏览器**才能使策略生效。

### 3. 代理配置
所有启动脚本默认使用 `socks5://127.0.0.1:7890`，请确保：
- ✅ Clash 或其他代理软件正在运行
- ✅ 代理端口正确（可通过参数修改）
- ✅ 代理模式设置为"规则模式"或"全局模式"

### 4. 兼容性问题
某些激进的隐私设置可能破坏网站功能，如遇问题：
- 调整 Shields/追踪保护级别
- 添加网站到白名单
- 临时禁用某些保护

### 5. 扩展冲突
- **Brave：** 不要安装 uBlock Origin，会与内置 Shields 冲突
- **其他浏览器：** 已配置扩展白名单，阻止其他扩展安装

### 6. Firefox 特殊配置
- user.js 会在每次启动时覆盖 prefs.js
- 如需修改配置，请编辑 user.js 而非 about:config
- Resist Fingerprinting 可能影响某些网站

### 7. Chromium 更新
Chromium 无内置更新器，需要手动更新或使用包管理器（如 Chocolatey）。

## 🔧 常见问题

### Q1: 配置脚本运行失败怎么办？
**A:** 确保：
1. 以管理员身份运行 PowerShell
2. 浏览器已正确安装
3. 没有其他程序占用注册表

### Q2: 浏览器启动后策略没有生效？
**A:** 
1. 重启浏览器
2. 访问 `chrome://policy` 或 `about:policies` 检查策略
3. 如果策略未显示，重新运行配置脚本

### Q3: 某些网站无法正常访问？
**A:** 
1. 检查代理是否正常运行
2. 尝试临时禁用追踪保护
3. 将网站添加到白名单

### Q4: 扩展没有自动安装？
**A:** 
1. 检查网络连接
2. 手动访问 `chrome://extensions` 查看错误信息
3. 重新运行配置脚本

### Q5: 如何为不同浏览器使用不同的代理？
**A:** 
```powershell
# 每个浏览器使用不同的代理端口
.\LAUNCH_CHROME.ps1 -ProxyPort 7890
.\LAUNCH_FIREFOX.ps1 -ProxyPort 7891
.\LAUNCH_BRAVE.ps1 -ProxyPort 7892
```

### Q6: 如何恢复默认配置？
**A:** 
1. 删除对应的注册表项
2. 删除浏览器配置文件
3. 重新安装浏览器

### Q7: Brave 的广告拦截效果不好？
**A:** 
1. 访问 `brave://adblock` 检查 Shields 状态
2. 确认 Shields 设置为"标准"或"激进"模式
3. 不要安装额外的广告拦截扩展

### Q8: Firefox 的 Resist Fingerprinting 影响网站功能？
**A:** 
1. 编辑 user.js，将 `privacy.resistFingerprinting` 设置为 `false`
2. 重启 Firefox
3. 或者使用 LibreWolf（已优化兼容性）
